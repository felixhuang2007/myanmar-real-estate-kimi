## 1. 项目简介

缅甸房产平台是一个完整的房产交易解决方案，支持C端买家、B端经纪人、平台管理后台多端协作。

### 1.1 核心业务
- **房源管理**: 发布、搜索、验真
- **ACN协作网络**: 5角色分佣机制
- **即时通讯**: 买家-经纪人沟通
- **交易闭环**: 看房预约 → 成交 → 分佣结算

## 2. 模块详细说明

### 2.1 用户服务 (03-user-service)
**职责**: 用户注册、登录、实名认证、账号管理

**核心功能**:
- 手机号注册/登录（短信验证码）
- JWT Token管理（Access/Refresh）
- 实名认证（身份证、护照）
- 用户画像管理

**数据表**:
- `users` - 用户基础信息
- `user_profiles` - 用户详细资料
- `user_verifications` - 实名认证记录
- `sms_verification_codes` - 短信验证码记录

### 2.2 房源服务 (04-house-service)
**职责**: 房源发布、搜索、管理

**核心功能**:
- 房源发布（支持图片/视频）
- 多维度搜索（位置、价格、房型）
- 房源验真流程
- 房源状态管理

**数据表**:
- `houses` - 房源主表
- `house_images` - 房源图片
- `house_videos` - 房源视频
- `house_audit_logs` - 审核记录

### 2.3 ACN服务 (05-acn-service)
**职责**: 经纪人协作网络、分佣计算

**核心功能**:
- 5角色分佣模型
- 成交订单管理
- 佣金结算计算
- 业绩统计

**ACN 5角色模型**:
```
房源方 (Property Side) 40%:
  - 录入人 (Entry Agent): 35%
  - 维护人 (Maintainer): 按比例分配
  - 转介绍 (Referrer): 按比例分配

客源方 (Client Side) 60%:
  - 带看人 (Viewer): 65%
  - 成交人 (Closer): 按比例分配

平台抽成: 10%
```

**数据表**:
- `agents` - 经纪人信息
- `companies` - 经纪公司
- `teams` - 团队
- `deals` - 成交订单
- `commission_records` - 分佣记录

### 2.4 预约服务 (06-appointment-service)
**职责**: 看房预约调度

**核心功能**:
- 预约创建/确认/取消
- 时间冲突检测
- 预约提醒
- 看房反馈

### 2.5 IM服务 (08-im-service)
**职责**: 即时通讯（预留接口）

**说明**: 当前为Stub实现，需接入第三方IM服务
**可选方案**:
- 环信 (Easemob)
- 融云 (RongCloud)
- 自研WebSocket服务

## 4. 数据库设计

### 4.1 核心实体关系

```
users (1) ────────< (N) user_profiles
      │
      ├───────────< (N) user_verifications
      │
      ├───────────< (N) user_favorites ───────> (N) houses
      │
      └───────────< (1) agents ──────────────> (N) companies

houses (N) ───────> (1) communities ───────> (1) districts
                                         └─> (1) cities

agents (N) ────────< (N) deals ────────────> (N) commission_records
      │
      └───────────< (N) team_members ───────> (N) teams
```

### 4.2 关键字段说明

**users表**:
- `id` - 自增主键
- `uuid` - 对外暴露ID（安全）
- `phone` - 手机号（唯一索引）
- `password_hash` - 密码哈希
- `status` - 状态: active/inactive/banned
- `created_at/updated_at/deleted_at` - 时间戳

**houses表**:
- `house_code` - 房源编码（唯一）
- `title` - 标题
- `price` - 价格
- `area` - 面积
- `bedrooms/bathrooms` - 房间数
- `status` - 状态: draft/pending/active/sold/offline

## 5. 接口设计

### 5.1 API规范
- 协议: HTTPS
- 格式: JSON
- 认证: JWT Bearer Token
- 版本: /v1/ 前缀

### 5.2 响应格式
```json
{
  "code": 200,
  "message": "success",
  "data": {},
  "timestamp": 1773798324,
  "request_id": "uuid"

```

### 5.3 错误码
| 代码范围 | 含义 |
|----------|------|
| 0-99 | 通用错误 |
| 100-199 | 用户模块 |
| 200-299 | 经纪人模块 |
| 300-399 | 房源模块 |
| 400-499 | ACN模块 |
| 500-599 | 预约模块 |
| 600-699 | IM模块 |

## 6. 安全设计

### 6.1 认证授权
- JWT双Token机制（Access + Refresh）
- Token过期时间: Access(24h) / Refresh(30d)
- 敏感操作需二次验证

### 6.2 数据安全
- 密码使用bcrypt加密
- 敏感字段加密存储
- 数据库连接SSL/TLS

### 6.3 接口安全
- 防SQL注入（参数化查询）
- 防XSS（输出编码）
- 限流保护
- CORS白名单

### 7.1 缓存策略

| 用户会话 | Redis | 24h |

### 7.2 数据库优化
- 常用查询字段加索引
- 分页查询限制最大页数
- 慢查询监控
- 读写分离（生产环境）
