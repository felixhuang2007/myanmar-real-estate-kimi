## 七、Bug追踪与回归

### 7.1 Bug生命周期

```
发现 → 记录 → 分配 → 修复 → 验证 → 关闭
 │       │       │       │       │
 │       │       │       │       └─ 回归测试
 │       │       │       └─ 开发修复
 │       │       └─ 项目经理分配
 │       └─ AI测试工程师记录
 └─ 测试发现
```

### 7.2 Bug严重度定义

| 级别 | 定义 | 示例 | 修复时限 |
|------|------|------|----------|
| 致命 | 系统崩溃/数据丢失 | APP闪退、无法启动 | 4小时 |
| 严重 | 核心功能不可用 | 无法登录、无法预约 | 8小时 |
| 一般 | 功能异常但可绕过 | 搜索不准确、UI错位 | 24小时 |
| 轻微 | 体验问题 | 提示语不当、图标模糊 | 48小时 |

### 7.3 回归测试策略

- **Bug修复后**：验证该Bug及相关功能
- **每日回归**：执行P0用例（自动化）
- **上线前回归**：全量P0+P1用例
