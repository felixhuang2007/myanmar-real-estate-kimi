## 附录：代码Review最佳实践

### 前端代码Review重点

1. **React Native / Flutter**
   - 组件拆分合理性
   - State管理规范性
   - 性能优化（memo/useMemo等）
   - 路由配置正确性

2. **HTML/CSS/JS**
   - 响应式布局
   - 兼容性处理
   - 资源懒加载
   - 代码分割

### 后端代码Review重点

1. **API设计**
   - RESTful规范
   - 接口幂等性
   - 版本控制
   - 文档完整性

2. **数据库**
   - 索引设计
   - 事务处理
   - 连接池配置
   - 分表分库策略

3. **缓存**
   - 缓存策略
   - 缓存一致性
   - 缓存穿透/雪崩防护

### 通用Review原则

- 每次Review不超过400行代码
- 使用工具辅助（SonarQube/ESLint等）
- 保持建设性态度
- 记录常见问题形成Checklist
