## 发现与修复的问题

### 问题1: API路由冲突

**描述**: Gin框架中`/houses/:id`参数路由与`/houses/my`静态路由冲突

**修复**: 调整路由注册顺序，静态路由先于参数路由注册

**代码变更**:
```go
// 需要认证的经纪人接口 - 必须先注册，确保静态路由优先匹配
agentHouses := r.Group("/houses")
agentHouses.Use(auth)
{
    agentHouses.GET("/my", c.GetMyHouses)  // 静态路由优先
    agentHouses.POST("", c.CreateHouse)
    agentHouses.PUT("/:id", c.UpdateHouse) // 参数路由在后
    // ...
}
```

### 问题2: 数据库连接失败

**描述**: API启动时数据库连接失败，db为nil导致所有业务模块被跳过

**根因**: 代码期望的环境变量名是`MYANMAR_PROPERTY_DATABASE_HOST`，但Docker容器设置的是`DB_HOST`

**修复**: 重新启动API容器，使用正确的环境变量名：
- `MYANMAR_PROPERTY_DATABASE_HOST=myanmar-property-db`
- `MYANMAR_PROPERTY_DATABASE_PORT=5432`
- `MYANMAR_PROPERTY_DATABASE_USER=myanmar_property`
- `MYANMAR_PROPERTY_DATABASE_PASSWORD=myanmar_property_2024`
- `MYANMAR_PROPERTY_DATABASE_DATABASE=myanmar_property`
- `MYANMAR_PROPERTY_REDIS_HOST=myanmar-property-redis`
- `MYANMAR_PROPERTY_REDIS_PORT=6379`
- `MYANMAR_PROPERTY_REDIS_PASSWORD=myanmar_redis_2024`

**验证**: 数据库连接成功后，日志显示：
```
[INFO] 数据库初始化成功
[INFO] initUserModule: user module routes registered
```

### 问题3: 测试脚本容器名称不匹配

**描述**: 测试脚本使用旧的容器名称（如`myanmar_postgres`）

**修复**: 更新为正确的容器名称：
- `myanmar-property-api` (不是`myanmar_api`)
- `myanmar-property-db` (不是`myanmar_postgres`)
- `myanmar-property-redis` (不是`myanmar_redis`)
