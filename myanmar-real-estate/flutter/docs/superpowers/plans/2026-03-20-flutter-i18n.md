# Flutter i18n Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add full internationalization to the Flutter Buyer and Agent apps supporting Myanmar (my, default), English (en), and Chinese (zh) using Flutter's official `gen-l10n` approach.

**Architecture:** `.arb` files in `lib/l10n/` define all UI strings; `flutter gen-l10n` generates a type-safe `AppLocalizations` class. A Riverpod `LocaleNotifier` persists the user's locale choice via `SharedPreferences`. First-time users see a language selection screen; locale is passed down via `MaterialApp.router`.

**Tech Stack:** Flutter 3.19, flutter_localizations (SDK), intl 0.18.1, flutter_riverpod 2.4.9, go_router 13.0.1, SharedPreferences (via LocalStorage utility)

**Spec:** `docs/superpowers/specs/2026-03-20-flutter-i18n-design.md`

import 'package:flutter/material.dart';
import 'package:flutter_riverpod/flutter_riverpod.dart';
import '../storage/local_storage.dart';

class LocaleNotifier extends Notifier<Locale> {
  @override
  Locale build() {
    final saved = LocalStorage.getLocale();
    return Locale(saved ?? 'my');
  }
