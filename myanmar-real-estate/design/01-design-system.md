> 文档版本: v1.0  
> 更新日期: 2026-03-17  
> 适用范围: C端APP / B端APP / Web后台 / 微信小程序

## 一、设计概述

### 1.1 设计理念

**"信赖·温暖·专业"**

缅甸房产平台的设计遵循以下核心原则：

- **信赖 (Trust)** - 通过清晰的视觉层级和真实的房源信息，建立用户对平台的信任
- **温暖 (Warmth)** - 运用缅甸文化色彩，营造亲切友好的用户体验
- **专业 (Professional)** - 简洁有序的布局，体现房产交易的专业性

### 1.2 设计目标

1. **降低认知负荷** - 让用户快速找到所需信息和功能
2. **建立信任感** - 通过验真标识、真实图片等元素建立安全感
3. **提升转化率** - 优化关键转化路径的交互体验
4. **适配本地化** - 完美支持缅语显示，符合缅甸用户习惯

### 1.3 平台适配说明

| 平台 | 设计密度 | 主要特点 |
|------|----------|----------|
| C端APP | 舒适型 | 大图展示、卡片式布局、情感化设计 |
| B端APP | 紧凑高效型 | 信息密集、表单为主、效率优先 |
| Web后台 | 桌面型 | 表格为主、数据可视化、管理操作 |
| 小程序 | 轻量型 | 功能精简、快速加载、即用即走 |

### 3.3 缅语排版特殊规则

#### 缅语文本特点

1. **字符高度** - 缅语字符比拉丁字符高，需增加行高
2. **无大小写** - 缅语没有大小写区分
3. **组合字符** - 存在大量组合字符，需确保字体支持
4. **基线位置** - 缅语基线较低，与英文混排时注意对齐

#### 排版适配规则

```css
/* 缅语内容增加行高 */
[lang="my"] {
  line-height: 1.8;  /* 比英文1.5更大 */
  letter-spacing: 0; /* 缅语不使用字间距 */
}

/* 中英缅混排 */
.mixed-text {
  line-height: 1.6;
  /* 英文/数字使用Noto Sans */
  /* 缅语自动使用Noto Sans Myanmar */
}

/* 价格显示 */
.price {
  font-family: 'Noto Sans Myanmar', sans-serif;
  font-weight: 700;
  font-feature-settings: 'tnum'; /* 等宽数字 */
}
```

#### 缅语适配示例

```
英文排版 (line-height: 1.5):
The quick brown
fox jumps over

缅语排版 (line-height: 1.8):
ကျွန်ုပ်တို့သည်
မြန်မာနိုင်ငံ၏
ပထမဆုံး
```

## 四、布局系统 (Layout)

### 4.1 栅格系统

#### 移动端栅格 (C端/B端 APP)

```
设计基准: iPhone 14 / 390px 宽度

栅格配置:
- 列数: 12列
- 列宽: 26px
- 间距: 8px
- 边距: 16px (左右)

可视化:
┌────────────────────────────────────────┐
│ marg │ col │gap│ col │gap│ col │ marg │
│ 16px │ 26px│8px│ 26px│8px│ 26px│ 16px │
└────────────────────────────────────────┘
总宽度: 16 + (26+8)*11 + 26 + 16 = 390px
```

#### 响应式断点

| 断点 | 宽度 | 设备类型 | 适配策略 |
|------|------|----------|----------|
| xs | < 360px | 小屏手机 | 紧凑布局，隐藏次要信息 |
| sm | 360-414px | 标准手机 | 默认设计基准 |
| md | 415-767px | 大屏手机 | 适当放大间距 |
| lg | 768-1023px | 平板 | 侧边导航，双列布局 |
| xl | 1024px+ | 桌面/Web | 最大容器1200px居中 |

#### Web后台栅格

```
容器宽度: 1200px (最大)
侧边栏: 200px (固定)
内容区: 1000px (自适应)

栅格配置:
- 列数: 24列
- 列宽: 32px
- 间距: 16px
- 边距: 24px
```

### 4.2 间距系统 (Spacing)

#### 基础间距单位: 4px

```
Spacing Scale:

xs:   4px   (极小间距 - 图标与文字)
sm:   8px   (小间距 - 紧凑元素)
md:   16px  (标准间距 - 卡片内边距)
lg:   24px  (大间距 - 区块间距)
xl:   32px  (极大间距 - 页面间距)
```

#### 常用间距组合

| 场景 | 间距值 | 应用 |

## 七、动效规范 (Motion)

### 7.1 动效原则

- ** purposeful (有目的)** - 每个动效都应有明确的功能目的
- ** subtle (微妙)** - 动效不应过于抢眼，干扰用户
- ** consistent (一致)** - 同类操作保持一致的动效
- ** performant (高性能)** - 使用CSS动画，避免重绘重排

### 7.2 时长规范

| 类型 | 时长 | 缓动函数 | 用途 |
|------|------|----------|------|
| 即时 | 100ms | ease-out | 按钮按下、图标切换 |
| 快速 | 200ms | ease-in-out | 悬停效果、小元素过渡 |
| 标准 | 300ms | cubic-bezier(0.4, 0, 0.2, 1) | 弹窗、浮层展开 |
| 舒缓 | 400ms | cubic-bezier(0, 0, 0.2, 1) | 页面切换、大元素 |

### 7.3 常用动效

#### 页面转场

```
类型: 滑动转场
方向: 从左向右 (进入) / 从右向左 (退出)
时长: 300ms
缓动: ease-in-out
```

#### 按钮交互

```
按下:
- scale: 1 → 0.96
- 时长: 100ms
- 缓动: ease-out

释放:
- scale: 0.96 → 1
- 时长: 100ms
- 缓动: ease-out
```

#### 列表加载

```
类型: 淡入+上滑
初始: opacity: 0, translateY: 20px
结束: opacity: 1, translateY: 0
时长: 300ms
交错: 每项延迟50ms
```

#### 弹窗展开

```
遮罩:
- opacity: 0 → 1
- 时长: 200ms

内容:
- opacity: 0 → 1
- scale: 0.9 → 1
- translateY: 20px → 0
- 时长: 300ms
- 缓动: cubic-bezier(0, 0, 0.2, 1)
```

#### 骨架屏加载

```
效果: 流光渐变
背景: linear-gradient(90deg, Gray-100 25%, Gray-200 50%, Gray-100 75%)
动画: background-position 从左到右循环
时长: 1.5s
```

### 7.4 性能优化

```css
/* 使用GPU加速 */
.animated-element {
  will-change: transform, opacity;
  transform: translateZ(0);
}

/* 避免触发布局重排 */
/* 只使用 transform 和 opacity 属性做动画 */
```
