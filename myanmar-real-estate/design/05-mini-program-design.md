> 文档版本: v1.0  
> 更新日期: 2026-03-17  
> 适用平台: 微信小程序  
> 目标用户: 购房者 / 租房者 (轻量化用户)

---

## 一、小程序产品概述

### 1.1 产品定位
微信小程序是缅甸房产平台的轻量化入口，针对"即用即走"场景设计，提供核心的房源浏览和咨询功能，降低用户使用门槛，便于社交分享传播。

### 1.2 设计目标
1. **轻量化** - 功能精简，只保留核心找房流程
2. **快速加载** - 首屏加载< 2秒，提升体验
3. **易于分享** - 支持房源卡片分享至微信聊天
4. **降低门槛** - 无需下载APP即可体验核心功能

### 1.3 与APP的设计差异

| 维度 | C端APP | 微信小程序 |
|------|--------|------------|
| 功能范围 | 全功能 | 核心功能 |
| 包体积 | 大 (50MB+) | 小 (2MB) |
| 用户路径 | 深度使用 | 快速浏览 |
| 登录方式 | 手机号/第三方 | 微信一键登录 |
| 消息通知 | APP推送 | 订阅消息 |
| 存储能力 | 大 | 有限 |

### 1.4 页面清单

| 模块 | 页面名称 | 优先级 | 说明 |
|------|----------|--------|------|
| 首页 | 首页 | P0 | 推荐房源入口 |
| 找房 | 房源列表 | P0 | 搜索筛选结果 |
| | 房源详情 | P0 | 房源信息展示 |
| 我的 | 我的页 | P0 | 个人中心 |
| 其他 | 授权登录页 | P0 | 微信授权 |

## 二、设计原则

### 2.1 轻量化原则

```
功能取舍标准:
✅ 保留: 房源浏览、搜索筛选、在线咨询、收藏、分享
❌ 去除: 地图找房、房贷计算、发布房源、IM聊天、预约管理

设计表现:
- 页面层级不超过3层
- 每页功能点不超过5个
- 优先使用原生组件
```

### 2.2 性能优先原则

```
加载优化:
- 图片懒加载
- 分页加载 (每页10条)
- 骨架屏占位
- 本地缓存优先

包体积控制:
- 图片CDN化
- 代码分包加载
- 精简第三方库
```

### 2.3 一致性原则

```
视觉一致:
- 沿用主设计系统色彩
- 字体规格保持一致
- 圆角/间距规范统一

交互一致:
- 下拉刷新统一
- 加载更多统一
- 返回按钮统一
```

   - 收藏: 跳转收藏列表

- 触发距离: 底部100rpx
- 加载提示: "正在加载..."
- 无更多数据: "没有更多了"

骨架屏:
- Banner区域灰色占位
- 房源卡片4个占位
- 带动画 shimmer 效果

## 十、性能优化规范

### 10.1 图片优化

```
图片规格:
- Banner: 750x300px, JPEG, 质量80%, < 100KB
- 房源封面: 400x250px, JPEG, 质量75%, < 50KB
- 头像: 200x200px, JPEG, 质量80%, < 20KB

图片加载:
- 使用 lazy-load 属性
- 使用 placeholder 占位
- 优先使用 CDN 加速
```

### 10.2 数据优化

```
分页加载:
- 首页推荐: 10条
- 列表页: 10条/页
- 详情相似推荐: 5条

数据缓存:
- 首页数据缓存5分钟
- 列表页数据缓存3分钟
- 详情数据缓存10分钟
```

### 10.3 代码优化

```
分包加载:
- 主包: 首页、列表、详情、我的
- 分包: 其他功能页面

预加载:
- 首页预加载列表页
- 列表页预加载详情页
```
